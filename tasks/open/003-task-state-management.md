# Task 003: Add Local Task State Management

Status: open

## Goal

Make the Kanban board interactive using local in-memory state.

## Scope

- Create tasks from the UI.
- Edit task title and description.
- Move tasks between statuses.
- Delete tasks.
- Validate required fields.

## Acceptance Criteria

- A user can create a task with a non-empty title.
- A user can edit an existing task.
- A user can move a task between all three columns.
- A user can delete a task.
- Invalid task input shows a clear validation message.
- Changes do not need to persist after reload yet.

## Deploy and Test Checkpoint

Deploy or run locally and manually verify create, edit, move, and delete interactions.
