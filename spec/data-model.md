# Data Model Specification

## Task

A task is represented as JSON:

```json
{
  "id": "task-123",
  "title": "Write deployment documentation",
  "description": "Describe local and Amplify deployment steps.",
  "status": "todo",
  "createdAt": "2026-05-27T13:00:00.000Z",
  "updatedAt": "2026-05-27T13:00:00.000Z"
}
```

## Fields

- `id`: Stable unique identifier generated by the backend or frontend.
- `title`: Required non-empty task title.
- `description`: Optional task details.
- `status`: One of `todo`, `in-progress`, or `done`.
- `createdAt`: ISO timestamp for task creation.
- `updatedAt`: ISO timestamp for the most recent change.

## Persisted File

The S3 object `tasks.json` stores the complete board state:

```json
{
  "tasks": []
}
```

## Validation

- Task titles must be trimmed and may not be empty.
- Unknown statuses must be rejected.
- Missing optional descriptions should be stored as an empty string.
- The app must handle a missing `tasks.json` object by initializing an empty task list.
